# Baseball Note プロジェクト概要

## このアプリの目的

Baseball Note は、野球の練習や試合で得た感覚を、その日のうちに短いメモとして残すためのWebアプリです。打撃・守備・走塁・投球・体調・フィジカルといった観点で記録し、あとから日付やキーワード、タグで振り返れることを目的としています。

サーバーやユーザー登録を必要とせず、同じブラウザ上ですぐ使える個人向けの記録ツールとして構成されています。

## 現在の仕様

この章に書かれている内容は、現在のアプリですでに実装されている仕様です。まだ実装が決まっていない構想は、後半の「今後の方向性」に分けて記載します。

### 主な機能

- 日付ごとのメモ作成と時刻順の一覧表示
- 前日・翌日ボタン、日付入力、今日ボタンによる日付移動（未来日は選択不可）
- 日別一覧の本文をタップまたはキーボード操作して、その場で編集
- 本文またはタグを変更した場合だけ表示される、一覧下部の保存・キャンセル操作
- 本文を空にして保存した場合と、編集状態から削除する場合の確認ダイアログ付き削除
- 「打撃」「守備」「走塁」「投球」「体調」「フィジカル」の複数タグ付与
- タグを選ばずにメモを保存することも可能
- 全期間を対象とした本文・タグ・日付のキーワード検索
- 検索画面での複数タグによる絞り込み（いずれかのタグに一致）
- 月曜日から日曜日までの保存済みメモを、日付見出しの下へ通常一覧と同じ2列構成でまとめる週間記録（画像は縦横比を維持したサムネイル表示）
- 週間記録での前週・次週移動、日付指定、メモから日別画面への移動（今週より未来へは移動不可）
- 1件のメモにつき画像1枚の添付と表示
- 添付画像の最大辺を1,400pxに縮小し、JPEG品質0.82で保存
- 全メモと画像をJSONファイルへ書き出すバックアップ
- JSONバックアップの読み込み（読込前に新規・上書き件数を案内し、同じIDのメモは確認後に上書き）
- バックアップ読込時のバージョン、必須項目、日付、ID、画像Data URL、タグの検証
- 日別一覧で時刻とタグを左列、本文を右列にまとめ、通常時の編集・削除アイコンを表示しないコンパクト表示
- PC向けサイドバーと、アプリ名・短縮日付・前後移動を1行にまとめたヘッダーとドロワーメニューを備えたスマートフォン向けレスポンシブ表示
- 初回だけ本文の直接編集を案内し、閉じた状態をメモデータとは別のブラウザ内設定へ保存
- Web App Manifest、アイコン、Service WorkerによるPWA対応
- GitHub Actionsを使ったGitHub Pagesへの自動ビルド・公開

### 保存方法

- メモと画像は、Dexieを介してブラウザ内のIndexedDBに保存されます。
- データベース名は `baseballNote`、テーブル名は `logs` です。
- 各メモにはID、対象日、作成日時、本文、画像、タグが保存されます。
- 画像はファイルそのものではなく、縮小・JPEG変換したBlobとしてメモ内に保持されます。
- サーバー保存やアカウント同期はありません。同じURLでも、ブラウザ・端末・ブラウザプロファイルが変わると別データになります。
- 端末間の移行やデータ保全には、アプリのバックアップ書き出し・読み込みを使用します。バックアップ形式はバージョン1のJSONで、画像はData URLとして含まれます。
- 未対応バージョンや不正なバックアップは理由を画面に表示して停止し、既存データへ書き込みません。過去データとの互換性のため、省略された画像とタグは空配列として補完します。

## 設計方針

今後の変更でも、次の方針を優先します。

- 中学生でも説明なしで迷わず使える、分かりやすい操作にする
- スマートフォンでの利用を優先し、PCでも使いやすい表示を保つ
- 記録までに必要な入力やタップをできるだけ少なくする
- 現在のシンプルなUIと落ち着いたデザインを維持する
- 通常画面ではメモ本文を主役にし、タグや検索は補助機能として扱う
- 入力時のタグは記録の整理と将来のAI分析に利用できる形で維持する
- 現段階では、共有サービスよりも個人利用での使いやすさを優先する
- 記録したデータを失わないことを重要視し、保存形式の互換性とバックアップ手段を守る

以下は、長期的に検討する「将来候補」です。現時点では実装決定済みの仕様ではなく、通常の開発TODOとは分けて扱います。実装する場合は、目的、利用場面、データの保存方法、使いやすさへの影響を確認してからTODOへ移します。

- **画像や動画を使った記録の拡張（将来候補）**: 現在の画像1枚の添付に加え、複数画像、動画、比較表示などを検討する
- **複数端末利用（将来候補）**: スマートフォンやPCの間で安全に同じ記録を利用できる同期方法を検討する
